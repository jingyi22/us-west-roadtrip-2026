# ✈️ 马来西亚航班姓名优化 UX 项目

## 项目概述
针对 Trip.com 马来西亚市场乘客姓名填写场景的 UX 优化项目，解决马来裔/华裔/印裔多元命名结构导致的姓名填错问题。

**背景数据**：
- 7,079 张机票中发现姓名问题
- 马来西亚是全球机票姓名修改率 Top 3 的市场
- 每月约 1,200 张姓名修改票

**核心问题**：
- "First/Last Name" 字段标签与马来西亚命名习惯不匹配（19.13% 姓名颠倒错误）
- 缺少 MRT 码（机读旅行证件）扫描功能
- 缺少多元文化姓名格式指引（马来裔/华裔/印裔）
- 付款前验证弹窗缺少拒绝登机风险提示

## 主要文件
- `MY_Name_Entry_Optimization_UX_Portfolio.html` — **最新 UX Portfolio 展示版**（含完整研究数据）
- `MY_Name_Optimization_Prototype.html` — 早期交互原型
- `Malaysian_Flight_Name_Optimization.pdf` — 原始 PDF 参考文档

## 改进点
1. **术语本地化** — "Given Name" / "Family Name" 替代 "First Name" / "Last Name"
2. **多元姓名示例** — 马来裔/华裔/印裔护照格式指引
3. **智能前缀验证** — bin/binti 性别错误实时检测
4. **MRZ 扫描** — 护照机读区扫码自动填表
5. **拒绝登机风险提示** — 付款前验证弹窗增加警告
6. **订单后姓名提醒** — 出票前免费修改窗口提示

## 屏幕
1. **Cover** — 研究数据展示（改签类型分布、错误发现时机、关键发现）
2. **Name Entry** — 改进后的姓名输入界面
3. **Verification** — Before/After 对比的验证弹窗
4. **Success** — 订单确认页 + 姓名修改提醒

## 研究方法
- 桌面研究：命名规范、航空规则、政府文档
- 竞品测试：10个平台（4家国际OTA、2家区域平台、4家航空公司）
- 内部分析：7,079张机票 · CPO数据 · 5场员工访谈
